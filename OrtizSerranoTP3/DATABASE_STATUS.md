# ✅ Base de Datos Creada Exitosamente con Hibernate

## 📊 Tablas Generadas Automáticamente

Hibernate ha creado las siguientes tablas en la base de datos H2:

### 1. **usuarios** (Tabla padre)
```sql
CREATE TABLE usuarios (
    id_usuario BIGINT GENERATED BY DEFAULT AS IDENTITY,
    apellido VARCHAR(100) NOT NULL,
    nombre VARCHAR(100) NOT NULL,
    email VARCHAR(255) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    PRIMARY KEY (id_usuario)
)
```

### 2. **estudiantes** (Herencia JOINED)
```sql
CREATE TABLE estudiantes (
    anio_ingreso INTEGER,
    id_usuario BIGINT NOT NULL,
    legajo VARCHAR(20) NOT NULL UNIQUE,
    carrera VARCHAR(100) NOT NULL,
    PRIMARY KEY (id_usuario),
    FOREIGN KEY (id_usuario) REFERENCES usuarios(id_usuario)
)
```

### 3. **docentes** (Herencia JOINED)
```sql
CREATE TABLE docentes (
    id_usuario BIGINT NOT NULL,
    departamento VARCHAR(100),
    especialidad VARCHAR(100),
    titulo VARCHAR(200),
    PRIMARY KEY (id_usuario),
    FOREIGN KEY (id_usuario) REFERENCES usuarios(id_usuario)
)
```

### 4. **administradores** (Herencia JOINED)
```sql
CREATE TABLE administradores (
    nivel_acceso INTEGER,
    id_usuario BIGINT NOT NULL,
    rol VARCHAR(50) NOT NULL,
    area VARCHAR(100),
    PRIMARY KEY (id_usuario),
    FOREIGN KEY (id_usuario) REFERENCES usuarios(id_usuario)
)
```

### 5. **materias**
```sql
CREATE TABLE materias (
    anio INTEGER,
    creditos INTEGER,
    cuatrimestre INTEGER,
    id_docente BIGINT,
    id_materia BIGINT GENERATED BY DEFAULT AS IDENTITY,
    codigo VARCHAR(20) NOT NULL UNIQUE,
    nombre VARCHAR(200) NOT NULL,
    descripcion TEXT,
    PRIMARY KEY (id_materia),
    FOREIGN KEY (id_docente) REFERENCES docentes(id_usuario)
)
```

### 6. **inscripciones** (Tabla intermedia Many-to-Many)
```sql
CREATE TABLE inscripciones (
    asistencias INTEGER,
    fecha_inscripcion DATE NOT NULL,
    nota_final FLOAT(53),
    nota_parcial1 FLOAT(53),
    nota_parcial2 FLOAT(53),
    id_estudiante BIGINT NOT NULL,
    id_inscripcion BIGINT GENERATED BY DEFAULT AS IDENTITY,
    id_materia BIGINT NOT NULL,
    estado VARCHAR(20) NOT NULL,
    PRIMARY KEY (id_inscripcion),
    FOREIGN KEY (id_estudiante) REFERENCES estudiantes(id_usuario),
    FOREIGN KEY (id_materia) REFERENCES materias(id_materia)
)
```

## 🔗 Relaciones Implementadas

1. **Usuario → Estudiante/Docente/Administrador**: Herencia JOINED (1:1)
   - FK: `id_usuario` en tablas hijas

2. **Docente → Materias**: One-to-Many (1:N)
   - FK: `id_docente` en tabla `materias`

3. **Estudiante → Inscripciones**: One-to-Many (1:N)
   - FK: `id_estudiante` en tabla `inscripciones`

4. **Materia → Inscripciones**: One-to-Many (1:N)
   - FK: `id_materia` en tabla `inscripciones`

5. **Estudiante ↔ Materia**: Many-to-Many a través de `inscripciones`
